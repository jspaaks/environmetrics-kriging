[h_array, gamma_array, count_array] = svgram_exp(pos,val)

This is the simplest form of calculating the semivariogram. Variable pos should be a NxM array, with N the number of observations and M the number of spatial dimensions. The Nx1 array val represents the corresponding value of the observed variable at location pos. Lags are aggregated into 10 bins ranging from 0 to the maximum observed lag. The function returns the bin midpoints in the output argument h_array; gamma_array represents the aggregated semivariance value at the bin midpoints. Variable count_array contains the number of observations in each bin.

[h_array, gamma_array, count_array] = svgram_exp(pos, val, parameter, value)

Additional parameter/value pairs can be included after the pos and val arrays. Variable parameter can be one of the following: 'cutoff', 'width' or 'nbins'. 'cutoff' sets the maximum lag to the corresponding value, 'width' sets the bin width to the corresponding value, and 'nbins' sets the number of bins to be used between 0 and the cutoff. If no cutoff has been specified, the number of bins is set to 10 by default. Multiple parameter/value pairs can be used, see example below.

Example:

[h,gamma,count] = svgram_exp(pos,val, 'cutoff', 1200, 'width', 100);

[h_array, angle_array, gamma_array, count_array] = semivar_exp(pos, val,...
'nbins_angle', value)

Inclusion of the parameter 'nbins_angle' invokes the "isotropy mode": the semivariance is not only sorted by lag h, but also by direction (from 0 to pi); the number of direction bins can be set by value. This implies that gamma_array and count_array become 2-D variables, the rows of which correspond to the bins of h_array; the columns correspond to the bins of angle_array.
