output
 variable name dimensions class description
input svmodel 2x1 cell Cell array with parameters of the semivariogram models. Format is:
svmodel{α,1}.c0 % nugget
svmodel{α,1}.c1 % c1
svmodel{α,1}.type % type of semivariogram model: 'Exp', 'Gau', or 'Sph'
svmodel{α,1}.range % range
svmodel{α,1}.longname % long identifier
svmodel{α,1}.shortname % short identifier used in gstat.exe command file (should be one word)

(Where α is either 1 or 2)

input xvmodel 1x1 cell Cell array with parameters of the cross-variogram model. Format is:
xvmodel{1,1}.c0 % nugget
xvmodel{1,1}.c1 % c1
xvmodel{1,1}.type % type of semivariogram model: 'Exp', 'Gau', etc.
xvmodel{1,1}.range % range
xvmodel{1,1}.longname % long identifier
xvmodel{1,1}.shortname % short identifier used in gstat.exe command file (should be one word)


input fnames 1x1 struct Structure array with filenames. Format is:
fnames.data{1} % filename of data file variable 1
fnames.data{2} % filename of data file variable 2
fnames.pred % filename of prediction locations file
fnames.cmd % filename of gstat.exe command file
fnames.out % filename of file to which the output is written



input parameter 1xN char Char array with one of the permissible parameter names (see table below). The value of "parameter" is specified in variable "value" immediately following "parameter". Multiple parameter/value pairs can be used. See examples below.

input value depends on "parameter" depends on "parameter" "value" determines the value of variable "parameter" immediately preceding it.

output G NxM numeric Output generated by gstat.exe.

Writes the gstat command file with the semivariograms specified in "svmodel" and the cross-variogram specified in "xvmodel", runs gstat.exe with it, and reads the results back into the MATLAB workspace.

Parameter name Description
'avgidentical' Averages measurements at the same location.
'cmdcomments' If 'cmdcomments' is set to 'on', some commentary lines are added to the *.cmd files to clarify the meaning of that part of the file.
'dims' Number of dimensions of location (for example, 'dims' is 2 for [x,y] data)
'max' Sets the maximum number of observations to use for a prediction (see figure).
'min' Sets the minimum number of observations to use for a prediction. If 'radius' is also specified and less than 'min' observation points are present within the search radius, a missing value is returned.
'misval' Missing value character(s)
'mode' 'mode' can be set to 'p', 'cs' or 'us' to indicate that a prediction, conditional simulation or unconditional simulation is wanted. Default is 'p'.
'pop_mean' Sets the population mean used in simple kriging and unconditional simulation
'radius' Sets the search radius within which observations are used for spatial prediction (see figure).
'showcmdfile' If 'showcmdfile' is set to 'on', the command file is shown in the MATLAB command window after it has been generated.
'xblocksize' Sets the blocksize in x-direction (to be used with block kriging)
'yblocksize' Sets the blocksize in y-direction (to be used with block kriging)
'zblocksize' Sets the blocksize in z-direction (to be used with block kriging)

Examples of how the function can be used:

G = mgstat_lite(svmodel,fnames, 'dims', 2,'radius',200)
G = mgstat_lite(svmodel,xvmodel,fnames, 'dims', 2,'min',20,'avgidentical','on')
